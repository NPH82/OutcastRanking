# OutcastRanking Deployment Guide

## Quick Deploy to Render.com (Free)

### Option 1: One-Click Deploy
1. Go to [render.com](https://render.com)
2. Sign up/login with GitHub
3. Click "New +" → "Web Service"
4. Connect your GitHub repository: `NPH82/OutcastRanking`
5. Render will auto-detect the `render.yaml` and deploy

### Option 2: Manual Setup
1. Fork or push this repo to GitHub
2. Go to [render.com](https://render.com) and create a new Web Service
3. Connect your repository
4. Configure:
   - **Name**: outcast-ranking
   - **Environment**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn run:app`
5. Click "Create Web Service"

Your app will be live at: `https://outcast-ranking.onrender.com`

## Database Persistence

⚠️ **Important**: Free tiers use ephemeral storage. Your SQLite database will reset on redeploys.

**Solutions**:
1. **Render PostgreSQL** (free addon) - Recommended
2. **Railway PostgreSQL** (included)
3. **Supabase** (free PostgreSQL hosting)

To switch to PostgreSQL, update `config.py`:
```python
SQLALCHEMY_DATABASE_URI = os.getenv('DATABASE_URL', 'sqlite:///rankings.db')
```

## Troubleshooting

**App crashes on startup?**
- Check logs in Render dashboard
- Verify `requirements.txt` has all dependencies
- Ensure `gunicorn` is installed

**Database not persisting?**
- Migrate to PostgreSQL (see above)
- Add database volume (paid feature)

**Slow performance?**
- Free tier spins down after inactivity
- First request after idle takes ~30 seconds
- Upgrade to paid tier for always-on
